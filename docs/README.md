## Keeping Docs Current

1. Update docs as part of the same pull request that changes behaviour.
2. Prefer small, concrete sections instead of sprawling encyclopedias.
3. When something becomes obsolete, delete it rather than letting it rot.
4. Use TODO markers sparingly and track open gaps in the issue tracker.
